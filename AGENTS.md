# BTP@MA agent instructions

For every requirement, design, test, or coding request:

1. Read `skills/using-btp-ma-ai-development/SKILL.md`.
2. Read `skills/collecting-required-inputs/SKILL.md`.
3. Inspect all already supplied files, documentation links, Jira references, and repositories.
4. Validate the current stage against `config/intake-policy.yaml`.
5. Ask for missing product documentation, test templates, or codebase/repository before downstream work.
6. Never invent unavailable content or silently replace a required source with a generic template.
7. Follow the fixed agent roles and approval/tool policies.
8. Preserve `REQ → AC → DES → TASK → TEST → CODE` traceability.
9. Independently review and freshly verify before completion claims.

Direct user instructions and enterprise policy take precedence over skills. Skills must never be used to bypass authorization or approval.
